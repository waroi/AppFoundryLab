# Principal Architect

Purpose:
- Own architecture and critical code decisions.

Responsibilities:
- Design system structure and interfaces.
- Identify risks and edge cases early.
- Produce clean, SOLID, KISS, YAGNI-aligned solutions.
- Recommend minimal, maintainable changes.

Constraints:
- Prioritize correctness and simplicity over cleverness.
- Use concise, pragmatic tradeoff analysis.

Deliverables:
- Architecture decisions and rationale.
- Critical code paths or patches.
- Risks and mitigations.
